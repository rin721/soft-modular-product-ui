# Content Rules

## 标题层级
- 每个页面只有一个 H1。
- H2 用于主要 section，H3 用于卡片组或组件标题。
- 标题应清楚表达结果或价值，不使用空泛口号。
- H1 建议 8-16 个汉字或 4-9 个英文词。
- H2 建议 6-14 个汉字或 3-8 个英文词。

## 副标题长度
- Hero 副标题 1-2 行。
- Section 描述 1-2 句。
- 卡片描述 1-2 行。
- 移动端优先删减副标题，而不是缩小字号。

## 段落密度
- 正文每段 40-90 个汉字或 20-45 个英文词。
- 长说明拆成短段或列表。
- 卡片内不放置超过 3 行正文。
- 信息条正文应简短，重要链接单独可点击。

## CTA 文案长度
- 主 CTA：2-6 个汉字或 1-4 个英文词。
- 次 CTA：2-8 个汉字或 1-5 个英文词。
- CTA 应描述动作结果，例如“创建工作区”“查看方案”“预约演示”。
- 避免多个同级 CTA 争夺注意力。

## 信息优先级
1. 用户能获得什么结果。
2. 该结果如何达成。
3. 关键功能或模块。
4. 可信度、数据或使用场景。
5. 下一步行动。

## 模块叙事顺序
- 官网首页：价值主张、功能、产品状态、信任信息、行动。
- Landing page：目标问题、解决方式、步骤、方案、常见问题、行动。
- 内容页面：分类、提示、内容列表、筛选和状态。
- 移动端页面：高频入口、当前状态、内容流、底部导航。

## 空状态文案
- 标题：说明当前为空。
- 说明：说明原因或下一步。
- CTA：提供创建、搜索、导入或返回。

示例：
```text
这里还没有内容
添加第一项后，它会显示在这个列表中。
```

## 表单提示文案
- Label 说明字段含义。
- Placeholder 只提示格式或示例，不替代 label。
- Help text 说明限制、格式或用途。
- 必填项用文本或符号说明，并确保辅助技术可读。

示例：
```text
项目名称
用于在列表和通知中识别这个项目。
```

## 错误提示文案
- 说明发生了什么。
- 说明如何修正。
- 避免责备用户。

示例：
```text
请输入有效的邮箱格式。
```

## 示例文案占位规则
- 使用中性组织、通用产品、虚构项目和占位数据。
- 不使用真实人物、真实组织、真实网址或不可授权口号。
- 数据示例使用范围、占位符或“自定义”。
- 图片说明使用“产品界面预览”“内容封面占位”“团队协作示意”等通用表达。

## 标题与容器匹配
- Hero 可以使用大标题。
- 卡片、侧栏、表单和 modal 使用紧凑标题。
- 小容器内不使用 hero 级字号。
- 多语言标题必须检查最长词是否溢出。
