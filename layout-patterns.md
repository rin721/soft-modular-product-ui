# Layout Grammar

## 页面总体框架
页面使用“稳定导航 + 浅色内容容器 + 模块化 section”的结构。导航可以固定在侧边、顶部或底部；内容应在最大宽度内居中或按工具型页面采用全宽流式布局。

常用框架：
- `Side Rail + Content`：桌面端左侧 56px 图标栏，右侧滚动内容。
- `Top Bar + Content`：桌面或平板端顶部 56-72px 工具栏，内容居中。
- `Mobile App Shell`：移动端顶部 56px 工具栏、底部 56px 主导航、中间滚动内容。
- `Marketing Flow`：顶部导航、hero、功能 section、可信度 section、CTA、footer。

## Section 节奏
- Section 垂直间距使用 `spacing.section.sm/md/lg`。
- 同一 section 内部使用标题组、内容组和操作组分层。
- 标题组与内容组间距为 `spacing.6` 到 `spacing.8`。
- 卡片组与 CTA 间距为 `spacing.6`。
- 内容型页面可缩短 section 间距，但不得让标题、卡片和 CTA 紧贴。

## Hero 模式
### 1. 顶部横幅型
- 用浅色背景、轻量几何图形和大标题建立第一视觉。
- 高度 desktop 140-360px，mobile 96-220px。
- 适合内容门户、社区、产品更新页。

### 2. 左文右媒体型
- 左侧标题、正文、CTA；右侧产品界面预览、抽象图形或卡片组合。
- desktop 使用 5:7 或 6:6 栅格；mobile 堆叠。
- 适合 SaaS、工具、企业产品。

### 3. 居中紧凑型
- 标题居中，副标题 1-2 行，CTA 置于标题下方。
- 下方露出卡片或内容网格。
- 适合 landing page 和简单官网。

### 4. Dashboard Preview 型
- 首屏使用半透明面板、卡片和图标按钮组合。
- 适合需要展示产品能力的页面。
- 预览区必须使用虚构或中性数据。

## 内容区模式
- `Announcement Strip`：信息提示条，图标 + 短标题 + 正文链接，适合公告、更新、引导。
- `Feature Cluster`：3-6 个功能卡片，图标 + 标题 + 说明。
- `Media Grid`：2-4 列卡片，图片 + 标题 + 元信息。
- `Text Panel`：窄容器长文，适合说明和 FAQ。
- `Split Narrative`：文本与卡片或媒体交替，适合产品故事。

## 卡片区模式
- Desktop 默认 3-4 列，最小卡片宽度 240px。
- Tablet 默认 2-3 列。
- Mobile 默认 2 列；长文本、价格方案和表单使用 1 列。
- 卡片可以无边框，用图片比例、标题层级和间距构成边界。
- 重点卡片可使用 `shadow.card.subtle`，普通卡片默认 `shadow.none`。

## CTA 区模式
- `Inline CTA`：位于 hero 或 section 标题组下，按钮不超过 2 个。
- `Panel CTA`：浅色或半透明面板，标题 + 说明 + CTA。
- `Footer CTA`：页脚上方的收束行动，适合长页面。
- CTA 区不使用过多装饰；主按钮和次按钮需有明确层级。

## 页脚模式
- 页脚使用浅色背景或无背景。
- 内容分为标识区、链接组、说明文本和辅助信息。
- 链接组 2-4 列，移动端堆叠。
- 页脚不得抢占主 CTA 的视觉重量。

## 栅格规则
- 基础内容容器：`container.content`。
- 宽屏视觉容器：`container.wide`。
- 窄文本容器：`container.narrow`。
- Desktop gutter 32-64px。
- Tablet gutter 24-32px。
- Mobile gutter 16px。
- 卡片网格使用 `repeat(auto-fit, minmax(min(100%, 240px), 1fr))` 或按输出模式映射。

## 容器宽度
- 普通内容：1040-1120px。
- 宽屏横幅：1180-1240px。
- 长文本和表单：640-760px。
- 侧栏固定宽度：56px。
- 移动顶部栏和底部栏高度：56px。

## 响应式重排方式
- 侧栏在移动端变为顶部工具栏和底部主导航。
- 横向标签在移动端可横向滚动，但必须有可见当前项。
- 双栏 hero 在移动端改为上下堆叠。
- 多列卡片在移动端优先 2 列；信息量较大时改为 1 列。
- 浮层菜单在移动端使用抽屉或底部 sheet，宽度 72-88vw。

## 常见页面模板
### 官网首页
导航 + hero + 功能卡片 + 产品预览 + 可信度模块 + CTA + footer

### Landing Page
导航 + 高聚焦 hero + 价值点 + 使用流程 + 方案卡片 + FAQ + CTA

### 内容门户
侧栏或顶部工具栏 + 横幅 + 分类标签 + 信息提示条 + 内容卡片网格

### 企业展示页
顶部导航 + 稳重 hero + 服务分区 + 数据模块 + 客户评价 + CTA + footer

### 移动端首页
顶部工具栏 + 信息提示条 + 双列卡片或单列功能块 + 底部导航

## 可替代布局组合
- Hero 可从顶部横幅型替换为左文右媒体型。
- 卡片网格可从媒体型替换为功能型、数据型或价格型。
- 导航可从侧栏替换为顶部栏，但当前项必须明显。
- 信息提示条可替换为浅色公告卡或 inline banner。
- 页脚可简化为一行版权与少量链接。

## 过拟合规避规则
- 不保留固定页面顺序；按任务目标选择模块。
- 不保留具体导航项、具体 CTA、具体图片组合或固定卡片数量。
- 不让装饰图形成为识别性符号。
- 不将单个断点或单个像素值写死为唯一实现。
- 不把内容门户布局机械套用到企业页或产品页。
