# Validation Checklist

## 美学风格
- [ ] 是否保持明亮、柔和、轻量、模块化。
- [ ] 主色是否只用于关键操作、选中、徽标、图标重点和少量装饰。
- [ ] 页面是否避免单一色系堆叠和过度装饰。
- [ ] 卡片、导航、信息条是否保持克制圆角和轻量阴影。

## Design Tokens
- [ ] 是否使用 `design-tokens.json` 的语义 token。
- [ ] 是否避免硬编码主色、阴影、圆角、间距和断点。
- [ ] 主色替换后是否同步 hover、focus、pressed、disabled、ripple 和 shadow。
- [ ] JSON token 是否结构完整且命名语义化。

## Layout Grammar
- [ ] 是否选择了合适页面框架。
- [ ] section 间距是否符合页面密度。
- [ ] hero 是否露出下一段内容的开头。
- [ ] 卡片网格是否可响应式重排。
- [ ] 是否避免固定复刻结构。

## Component Recipes
- [ ] 是否包含所需组件。
- [ ] 每个组件是否有用途、结构、token、状态、响应式和可访问性规则。
- [ ] 按钮、导航、卡片、表单、modal、footer、CTA 是否符合 recipe。
- [ ] 组件示例是否中性、可替换。

## Interaction States
- [ ] 是否包含 hover。
- [ ] 是否包含 active。
- [ ] 是否包含 focus-visible。
- [ ] 是否包含 disabled。
- [ ] 是否包含 loading。
- [ ] 是否包含 error。
- [ ] 是否包含 success。
- [ ] 是否包含 empty。
- [ ] 是否包含 keyboard 和 touch 行为。
- [ ] 是否支持 reduced motion。

## Responsive Rules
- [ ] 是否覆盖 desktop。
- [ ] 是否覆盖 tablet。
- [ ] 是否覆盖 mobile。
- [ ] 是否定义容器宽度、gutter、栅格变化和卡片重排。
- [ ] 移动端触控目标是否不小于 44x44px。
- [ ] 移动端底部导航是否避免遮挡内容。
- [ ] 长文本是否有折行策略。

## Accessibility
- [ ] 文本对比度是否满足 WCAG AA。
- [ ] 可交互元素是否有 focus-visible。
- [ ] 键盘是否可操作。
- [ ] 表单是否有 label、帮助文本和错误关联。
- [ ] 图片是否有 alt 或被标记为装饰。
- [ ] 标题层级是否合理。
- [ ] 颜色是否不是唯一信息载体。
- [ ] 动效是否可关闭或降级。

## Content Rules
- [ ] H1 是否唯一且明确。
- [ ] 副标题是否不超过 2 行。
- [ ] 段落是否可扫描。
- [ ] CTA 文案是否短而具体。
- [ ] 空状态、表单提示和错误提示是否友好且可行动。
- [ ] 示例文案是否避免真实人物、组织、网址和不可授权口号。

## 可复用性
- [ ] 产物是否能被 AI agent 独立读取和执行。
- [ ] 是否不依赖专有插件或私有协议。
- [ ] 是否可扩展 token、组件和布局。
- [ ] 是否保留适配主色、字体、圆角、密度和动效的能力。

## 发布清洁度
- [ ] 是否不包含真实网址。
- [ ] 是否不包含具名商业标识。
- [ ] 是否不包含专有图片、专有图标、不可授权文案和真实业务数据。
- [ ] 是否不包含过程叙述。
- [ ] 是否不包含固定复刻顺序。
- [ ] 是否只保留第一方规则、语义 token、中性示例和质量标准。
